# Plan реализации системы KIM

## Обзор системы

**KIM** — AI-система анализа социальной политики на основе мультиагентной архитектуры с использованием Claude (Anthropic). Система выявляет ключевые социальные проблемы в городах, анализирует пробелы законодательства и генерирует предложения инициатив.

---

## Последовательный план реализации

### 1. Инициализация проекта и настройка окружения

Создание структуры Python-пакета, конфигурация инструментов разработки, настройка зависимостей и точки входа в пакет.

### 2. Разработка моделей данных (Data Models)

Определение всех Pydantic-моделей для описания доменных сущностей системы: перечисление отраслей социальной политики, модели проблем города, пробелов законодательства, инициатив и итогового отчёта.

### 3. Реализация специализированных агентов

Разработка трёх специализированных AI-агентов, каждый из которых выполняет конкретную аналитическую задачу через вызов Claude API:
- `ProblemIdentifierAgent` — выявление социальных проблем
- `LegislativeAnalystAgent` — анализ пробелов законодательства
- `InitiativeGeneratorAgent` — генерация инициатив

### 4. Реализация оркестратора

Разработка `OrchestratorAgent`, который координирует работу специализированных агентов, управляет единым клиентом Anthropic и формирует итоговый отчёт.

### 5. Разработка CLI-интерфейса

Создание командного интерфейса на базе Typer с командами `analyze` и `list-sectors`, поддержкой вывода через Rich и экспортом результатов в JSON.

### 6. Написание тестов

Покрытие кода модульными тестами с мок-объектами для агентов и тестами валидации моделей данных. Обеспечение возможности запуска тестов без реального API-ключа.

### 7. Документация и финальная конфигурация

Написание README с описанием архитектуры, установки, использования и примерами. Финализация `pyproject.toml` с точками входа CLI.
